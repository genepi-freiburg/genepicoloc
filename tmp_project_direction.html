GWAS Colocalization Project - Technical Direction

Project Goal

Public-facing showcase for an R package for GWAS colocalization analysis, deployed on personal VPS infrastructure.

Architecture

yourname.de        → Astro static site (landing page, docs, project info)
app.yourname.de    → Shiny app (interactive colocalization analysis)

Both served via Nginx, SSL via Let’s Encrypt, admin access via WireGuard VPN.

Decision: No frontend framework inside Shiny

Considered adding TypeScript/Astro directly into Shiny but decided against it. Shiny is a data exploration tool, not a website builder - mixing frameworks creates unnecessary complexity. Clean separation is the right approach.

Stack by layer

Astro site (yourname.de)

  • Static site generation - fast, minimal JS sent to browser
  • TypeScript for any interactive elements
  • Project description, method explanation, links to Shiny app and R package
  • Impressum + Datenschutz (use personal work/institute address)

Shiny app (app.yourname.de)

  • Core colocalization analysis and results
  • Interactive plots via LocusZoom.js for regional association plots
  • Scrollable genome-wide Manhattan via GenomeSpy
  • Both embedded via htmlwidgets or direct JS integration
  • AI interpretation of coloc results via Claude API (server-side, key never exposed to frontend)

AI interpretation layer

  • Shiny passes coloc statistics to Claude API
  • Claude returns plain-language interpretation of results
  • e.g. “Strong evidence of shared causal variant between trait A and B at locus X…”
  • Goal: publication-style results paragraph output
  • Consider letting users supply their own API key for public deployment

Learning path alongside this project

TypeScript basics (1-2 weeks)
  → Astro (build the landing page as first real project)
    → JS visualization libraries (LocusZoom.js / GenomeSpy)
      → Docker cleanup of full deployment

Infrastructure

  • Personal domain + multiple VPS machines (already running)
  • WireGuard VPN connecting VPS + home GPU server
  • Home GPU server available for heavy compute if needed
  • Avoid university hospital IT (too restrictive for public-facing projects)

Key libraries to explore

Library Purpose

GenomeSpy Scrollable genome-wide Manhattan, GPU accelerated
